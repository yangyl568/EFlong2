## 为什么使用vite 比 webpack 快很多？

Vite 比 Webpack 快，主要原因有以下几点：

- **原理不同**  
  Vite 利用现代浏览器原生支持 ES Modules（ESM），开发时不需要像 Webpack 一样“打包”所有模块，只在浏览器请求时即时按需提供。

- **冷启动极快**  
  Vite 直接用原始源码启动开发服务器，只对 `index.html` 和被浏览器请求的 JS/TS 文件做“即时编译”，初次启动几乎不需要打包全量代码。

- **按需编译**  
  只有你访问的页面相关的文件才会被 Vite 处理和转译，未访问的代码不会被编译；Webpack 启动时会把所有依赖都打包一遍。

- **依赖预构建**  
  Vite 会把 node_modules 里的依赖（第三方库）用 esbuild 预先编译好，速度极快（esbuild 用 Go 写的，比 JS 实现的 Webpack 快几十倍）。

- **HMR（热更新）更快**  
  Vite 只热更新实际变动的模块，不需要像 Webpack 那样重构整个模块图，反馈极快。

---

**简要总结：**  
Vite 利用浏览器原生 ESM + esbuild 预编译 + 按需即时编译，极大提升了开发时的启动和热更新速度，而 Webpack 需要全量打包和复杂的模块处理，速度慢很多。
